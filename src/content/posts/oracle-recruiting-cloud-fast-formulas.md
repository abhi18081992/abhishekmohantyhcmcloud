<p style="font-size:16px;margin-bottom:18px;color:#000;font-family:'Open Sans',sans-serif;">This formula loops through all prescreening answers for a job application. If the candidate actually answered at least one question (not just skipped it), the formula returns Y. This is used to auto-advance candidates who completed the prescreening questionnaire.</p>

<div style="background:#1a1a1a;border-radius:6px;padding:18px 20px;margin:20px 0;font-family:Consolas,'Courier New',monospace;font-size:14px;color:#ccc;line-height:1.8;overflow-x:auto;white-space:pre-wrap;word-wrap:break-word;"><span style="color:#6b8e6b;font-style:italic;">/* Prescreening Answer Validation
   Returns Y if the candidate answered at least one question.
   Uses WAS_DEFAULTED to distinguish real answers from defaults. */</span>

<span style="color:#e67e22;">DEFAULT_DATA_VALUE FOR</span>
  IRC_CSP_JOBAPP_PRESCREEN_RESPONSE_ANSWER_CODE <span style="color:#6cacec;">IS</span> <span style="color:#8bc48b;">' '</span>
<span style="color:#e67e22;">DEFAULT_DATA_VALUE FOR</span>
  IRC_CSP_JOBAPP_PRESCREEN_RESPONSE_QSTN_CODE <span style="color:#6cacec;">IS</span> <span style="color:#8bc48b;">' '</span>

CONDITION_RESULT  = <span style="color:#8bc48b;">'N'</span>
CONDITION_MESSAGE = <span style="color:#8bc48b;">''</span>

j = <span style="color:#d4a76a;">1</span>
<span style="color:#e67e22;">WHILE</span> IRC_CSP_JOBAPP_PRESCREEN_RESPONSE_QSTN_CODE.<span style="color:#6cacec;">EXISTS</span>(j) <span style="color:#e67e22;">LOOP</span>
(
  <span style="color:#e67e22;">IF</span> (<span style="color:#6cacec;">WAS_DEFAULTED</span>(IRC_CSP_JOBAPP_PRESCREEN_RESPONSE_ANSWER_CODE[j]) = <span style="color:#8bc48b;">'N'</span>) <span style="color:#e67e22;">THEN</span>
  (
    CONDITION_RESULT = <span style="color:#8bc48b;">'Y'</span>
    CONDITION_MESSAGE = <span style="color:#8bc48b;">'Prescreen answered at question '</span> || <span style="color:#6cacec;">TO_CHAR</span>(j)
    <span style="color:#e67e22;">RETURN</span> CONDITION_RESULT, CONDITION_MESSAGE
  )

  j = j + <span style="color:#d4a76a;">1</span>
)

CONDITION_MESSAGE = <span style="color:#8bc48b;">'No prescreen response found after '</span> || <span style="color:#6cacec;">TO_CHAR</span>(j - <span style="color:#d4a76a;">1</span>) || <span style="color:#8bc48b;">' questions'</span>
<span style="color:#e67e22;">RETURN</span> CONDITION_RESULT, CONDITION_MESSAGE</div>

<div style="font-size:20px;font-weight:700;color:#000;margin:30px 0 14px;font-family:'Open Sans',sans-serif;">How this formula works</div>

<p style="font-size:16px;margin-bottom:14px;color:#000;font-family:'Open Sans',sans-serif;"><strong>DEFAULT_DATA_VALUE</strong> — Prescreening DBIs are arrays. When a candidate skips a question, Oracle doesn't leave the field null. It fills it with the default value you specify here (a space character). This is important for how we check the data later.</p>

<p style="font-size:16px;margin-bottom:14px;color:#000;font-family:'Open Sans',sans-serif;"><strong>WHILE .EXISTS(j) LOOP</strong> — Loops through each prescreening question. Index 1 is the first question, index 2 is the second, and so on. When .EXISTS returns FALSE, we've gone past the last question and the loop ends.</p>

<p style="font-size:16px;margin-bottom:14px;color:#000;font-family:'Open Sans',sans-serif;"><strong>WAS_DEFAULTED() = 'N'</strong> — This is the key check. WAS_DEFAULTED returns 'Y' if the value came from the DEFAULT (meaning the candidate skipped the question), and 'N' if the value is real data (meaning the candidate actually answered). When it returns 'N', we know we have a genuine response.</p>

<p style="font-size:16px;margin-bottom:14px;color:#000;font-family:'Open Sans',sans-serif;"><strong>Early RETURN</strong> — The moment we find one valid answer, we return Y immediately and stop. No need to scan the rest. The message tells you exactly which question had the response — useful when a recruiter asks "did the formula actually check the answers?"</p>

<p style="font-size:16px;margin-bottom:18px;color:#000;font-family:'Open Sans',sans-serif;"><strong>Fallback RETURN</strong> — If the loop finishes without finding any real answer, the formula returns N. The message confirms how many questions were checked, so you can verify the formula ran through the entire questionnaire and didn't exit early by mistake.</p>
